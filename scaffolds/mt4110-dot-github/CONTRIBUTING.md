# Contributing

Thanks for contributing.

## Before You Start

- read the repository `README.md` and any repo-local docs before making changes
- use [SUPPORT.md](./SUPPORT.md) when you are not sure which path fits your question
- use GitHub Discussions first for broad ideas or usage questions when Discussions is enabled
- open an issue first for larger concrete changes
- keep pull requests focused
- prefer the existing API and design language over new abstraction layers

Please follow [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) in issues, pull requests, and review.

## Local Setup

Follow the setup and verification steps documented in the repository itself.

The shared default does not assume one package manager, build tool, or test command.

## Pull Requests

Please make sure your PR:

- uses a short descriptive branch name and a neutral PR title
- explains the user-facing reason for the change
- keeps docs in sync when public behavior changes
- adds or updates tests and verification where appropriate
- notes breaking changes clearly
- follows any repo-local release docs when the change affects release work

Avoid tool or assistant name prefixes in branch names, PR titles, and release titles.
If the repository provides local issue forms, pull request templates, or a repo-local `CONTRIBUTING.md`, the local guidance wins.
